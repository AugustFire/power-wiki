# power-wiki 产品审视

**审视时间**:2026-07-15
**项目阶段**:开发打磨期(对标 Confluence,内部使用)
**审视范围**:空间、导航、编辑器、阅读、生命周期、协作、权限
**不在范围内**:全文检索、Comment 功能、Reaction 表情

**空间模型口径**(更新于本轮反馈):

- **空间 = 团队**:一个团队对应一个空间,典型单用户所属空间 ≤ 5。空间下拉切换器已经足够日常使用,**不需要**做"收藏空间"或"空间总览页"作为高频入口。
- **个人空间 = 私人空间**:默认只对自己可见,管理员可浏览(用于运维/找回内容),作者可主动把页面"发布到团队空间"让其他人看。**不需要**做"分享给指定用户/组"路径。
- **没有"草稿"概念**:个人空间天然承担草稿角色(私写 + 预览),作者在团队空间里写的每一页都是协作/实时生效,**不**支持"先藏着"的状态切换;团队空间页没有"草稿 vs 已发布"两态。

每模块固定三段:**现状** — **问题** — **建议方向**。末尾附优先级汇总矩阵。

## 模块 1:空间 (Spaces)

### 现状

`spaces` 表(`apps/api/src/db/schema.ts`)记录一个空间的姓名/描述/色块/图标/类型(`personal` 或 `shared`)/拥有者。前端有三个地方显示空间身份:

- 顶栏 `SpaceSwitcher.vue`(团队空间下拉)
- 左侧 `Sidebar.vue` 的空间徽标(总是显示当前空间)
- 用户菜单里"我的空间"入口(跳个人空间)

管理员在 `/manager/spaces` 统一管空间、给组授权(`space_group_access`)。每个用户一个个人空间,默认只自己看;管理员可浏览个人空间,作者可主动"发布到团队空间"。

当前空间 ID(`activeSpaceId`)存 localStorage,登出时清,防止用户间残留。

**本模块适用口径**:空间 = 团队容器,典型单用户所属空间 ≤ 5;个人空间 = 私人空间(见文档顶部口径)。所以本模块的"目录化"诉求弱于 Confluence 这种大型组织场景,但"识别度/可读性"诉求不变。

### 问题

- **空间的"描述"字段填了用户看不见**。管理员创建空间时写了描述,但侧栏、切换器、空间首屏都没有任何地方展示这条描述。
- **空间视觉身份在三个地方长得不一样**:顶栏显示「色块+图标+首字母」,侧栏徽标只显示「色块+图标」,切换器下拉显示「色块+图标+首字母」,你切到不同地方会怀疑"是同一个空间吗"。
- **空间没有"归档"状态**,只能删除。删完内容就没了,不像 Confluence 可以保留只读状态;团队改组/重组时这是个常见诉求。
- **(次要)没有空间总览页**。典型 ≤ 5 个空间时下拉已经够用,但如果未来要展示"按最近活跃排序的空间卡片 + 各空间统计",需要一个轻量的着陆页。可后置。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P1 | 统一空间视觉身份(色块+图标+名字,三处一致) | 小 |
| P1 | 空间描述在切换器 tooltip + 空间首屏两处露出 | 小 |
| P2 | 空间归档状态(保留在切换器列表,标记只读,内容不丢) | 中 |
| P3 | 轻量 `/spaces` 总览页(可选,作为未来扩展点) | 中 |

## 模块 2:侧栏与页面树

### 现状

左侧 `Sidebar.vue` 展示当前空间的页面树,数据从 `usePagesStore` 拉、按 `parentId` 父子关系组装成树。懒加载:展开某个节点才请求它的子节点。

树的操作:

- 拖拽改顺序 / 拖拽改父子关系(下落区域分上/中/下三段:放兄弟前、放为子页、放兄弟后)
- 右键菜单:打开新窗口、复制链接、重命名、加同级/子页、复制、删除、发布到、提升为根
- 当前页面会自动展开祖先链并滚动到可见

展开状态按空间分开存,滚动位置也按空间存,刷新不会丢。

### 问题

- **没有"一键展开所有"和"一键收起所有"**,树很深的时候要点十几下 caret。
- **没有键盘导航**(↑↓ 上下、J/K 翻行、Enter 打开)。要快速在树里找页只能鼠标。
- **折叠的节点不显示里面有多少子页**。你看到一个折叠节点,不知道点开是 1 页还是 50 页。
- **没有多选 + 批量移动**。想把 20 个页一次性换个父,得拖 20 次。Confluence 支持 Shift 多选 + 批量移动。
- **没有 Tab/Shift+Tab 缩进调整**(Notion 风格),只能在树里拖。
- **拖动源节点没有可见轨迹**,只目标行有提示,深嵌套时容易跟丢。
- **排序字段是整数 `sortOrder`**,每次重排都要重算邻居排序号,数据量大时(单个父下 100+ 子)会成为性能瓶颈。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 折叠子节点徽章显示子页数(例:`▶ 设计文档 (8)`) | 小 |
| P1 | 一键展开/收起全部按钮 | 小 |
| P1 | 树键盘导航(↑↓ J/K Enter) | 中 |
| P1 | Shift 多选 + 批量移动到父/删除 | 中 |
| P2 | Tab/Shift+Tab 在树里缩进出缩进 | 中 |
| P2 | 拖动源节点加可见占位(横线/半透明原位置) | 小 |
| P3 | 排序换成分数字段(LexoRank 类),扛 100+ 同级 | 大(可后置) |

## 模块 3:首页与入口

### 现状

`/`(`HomeView.vue`)根据当前空间类型走两个分支:

- **团队空间**:欢迎横幅 → 4 张统计卡片(总数/今日编辑/本周编辑/我的页数) → 快捷动作 → "我最近访问" → "最近编辑" → 所有根页面网格
- **个人空间**:直接走 `MeDashboardView`(`/me` 路由,「我的工作」三段式:@提到我 / 个人空间草稿 / 我创建的页)

最近访问用 localStorage 的 `useRecentPages` 组合式记录(每用户、每设备)。**注意**:数据库里有 `user_recent_pages` 表,设计上是为了跨设备同步,但前端目前没接。

`/me/watched` 是被关注页面列表,`/activity` 是全工作区编辑历史。

### 问题

- **"最近访问"两端漂移**:localStorage 写一份,数据库存一份,前端只用前者。换浏览器历史就丢。
- **没有"收藏/星标"页面**,只有"被关注"。"我看这个页很多次但不想订阅它的更新"这种需求没出口。
- **首页统计只是几个死数字**(总数/今日/本周),没有"你离开后改了哪些"、"同事在讨论什么"。
- **首页组件文件自己承认脆弱**:`HomeView.vue` 文件级注释明说"团队空间/个人空间两套渲染挤在一个组件,需要稳定的包装 div 拆开"。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 把 `user_recent_pages` 表前端接上,跨设备同步最近访问 | 小 |
| P1 | 加"收藏/星标"页面(独立于 watch),侧栏顶部 Pinned 区 | 中 |
| P1 | 首页加"离开后变更"卡片:订阅列表/被关注页/创建页的变化 | 中 |
| P2 | 把 HomeView 拆成稳定的 wrapper + 两块 panel(团队/个人),去掉条件渲染 | 小 |

## 模块 4:路由与 URL

### 现状

用 hash 路由(`createWebHashHistory`),URL 形如 `http://host/#/p/V1StGXR8...`。路由表:

```
/login            登录
/reset-password   改密
/                 首页
/me               我的工作
/p/:id            阅读
/p/:id/edit       编辑
/p/:id/history    版本历史
/new              新建页
/activity         全局活动
/me/watched       关注列表
/manager/*        管理后台
```

页面用 nanoid 标识,**标题不出现在 URL 里**(标题纯展示)。空间不在 URL,靠 store 里的 `activeSpaceId` 隐式决定。

权限守卫链:登录态初始化 → 公开页放行 → 强制改密 → 管理后台校验 → 需登录。404 用 `NotFoundView`。

### 问题

- **URL 是一串乱码**:页标题叫"Q3 Planning",你截个图发群里,链接是 `/#/p/V1StGXR8_Z5jHi3B`。人读不出来,搜索也搜不到。
- **URL 里没有空间**:不能直接发"给我看下工程团队空间",只能发"给我看下这页",对方得手动切到对应空间。
- **没有版本 permalink**:历史页能看 v12 长啥样,但没法发"看一下 v5 那时候的样子"。
- **进入编辑就是提交式**,没有"预览/分屏"。一点编辑就要么 save 要么 cancel。
- **404 一种文案盖所有**:"这页不存在"、"你没权限"、"这页已被永久删除" 在用户看来没区别,后者其实可以提示"是否恢复"。
- **hash 路由的副作用**:URL 不会出现在服务端,不能 server-render 兜底,刷新总要走 SPA。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | URL 加可读 slug(`/p/:id-:slug` 或 `/p/:slug`),标题自动派生 slug 字段 | 中 |
| P1 | URL 加空间段(`/space/:key/p/...`),路由驱动 activeSpaceId | 中 |
| P1 | 版本 permalink(`/p/:id/history?v=N`),可单独打开某一版 | 小 |
| P2 | 编辑模式加"预览/分屏"切换 | 大 |
| P2 | 404 文案分三档:不存在 / 没权限 / 已永久删除(后者带恢复提示) | 小 |
| P3 | 路由从 hash 迁 history(配 nginx try_files) | 中(纯前端可后置) |

## 模块 5:编辑器核心与排版

### 现状

编辑器壳 `RichEditor.vue` 用 Tiptap。注册的扩展:`StarterKit`(heading/codeBlock 替换成自定义版)+ `TaskList/TaskItem` + `Table`(含 colwidth/背景色/对齐/垂直对齐)+ `CodeBlockLowlight`(带语言切换+复制按钮)+ `DragHandle`(hover 出现)+ `Placeholder` + `Link`(自动识别+粘贴自动转)+ `Color`+`TextStyle`+`Highlight` + `TextAlign` + `Callout` + `Toggle` + `PageRef` + `DateInline` + `Mention` + `ImageAttachment`。

气泡菜单:选中文字后弹粗体/斜体/删除线/行内代码/链接。
Markdown 输入规则:输入 `## ` 自动变 H2,`**` 变粗体,`- ` 变无序列表,等等。
`Cmd/Ctrl+S` 拦截防止浏览器保存网页弹窗,其他 Tiptap 默认快捷键全开。

### 问题

- **没有字号、字体选择器**(Tiptap 扩展 `FontSize`/`FontFamily` 没装),只能靠标题级别区分大小,层级表达受限。
- **没有格式刷**(复制某段的格式到其他段),格式复用只能手动重设。
- **Cmd+F 走浏览器原生查找**,搜不到 mark/节点里的文本,长文档查找不准。
- **`CheatSheetModal.vue` 存在但没接全局快捷键**(`?` 唤不出来),用户只能从工具栏 tooltip 一点点学。
- **没有阅读时长估算**(只有字数);也没有每节点字数统计。
- **`@` 提到人只能在空间成员里搜**,跨空间成员无入口。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 接 Tiptap `FontSize` + `FontFamily` 扩展,工具栏加选择器 | 小 |
| P0 | 全局 `?` 唤出 CheatSheetModal | 极小 |
| P1 | 编辑器内 Cmd+F 替换为 ProseMirror-aware 查找替换 | 中 |
| P1 | 格式刷(选段 → 抓 mark → 涂到目标段) | 中 |
| P2 | byline 加阅读时长估算(中英文不同语速) | 极小 |
| P2 | 跨空间 `@` 提及(管理员/空间成员可见时) | 中 |

## 模块 6:斜杠菜单与工具栏

### 现状

`SlashMenu.vue` 触发条件:在新一段开头敲 `/`(空白边界)。**15 个插入项全在一个平铺列表里**,没分组:

标题 1/2/3、列表、任务列表、表格、代码块、引用、分隔线、Callout、Toggle、图片/附件、页内引用、日期、@提及。

工具栏 `EditorToolbar.vue`:段落类型下拉、粗体/斜体/删除线/行内代码、链接弹窗、字体色、高亮色、列表、缩进/反缩进、对齐、图片/附件、表格、分隔线、代码块、Callout(含变体)、Emoji、日期。

DragHandle 是 hover 出现的小条,**只用来拖排序**,不能点出插入菜单。

### 问题

- **斜杠菜单 15 项全平铺**,没分组(基础/媒体/高级),没"最近使用"置顶,新增项越多越翻不动。
- **没有 Notion 风格的悬浮 "+" 按钮**(空 block hover 弹插入菜单)。新用户不知道可以插东西。
- **工具栏无溢出机制**,如果再加项,1280px 视口下可能挤压或溢出。
- **Markdown 输入规则没提示**,用户敲 `## ` 才会变标题,但没人知道有这个捷径。
- **缺一些常用插入**:水平线已经在了但斜杠菜单没暴露;状态标签(STATUS PILL);数学公式;视频嵌入;画板。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 斜杠菜单分组(基础 / 媒体 / 高级)+「最近使用」置顶 | 小 |
| P0 | 空 block hover 浮出 Notion 式 "+" 按钮,点击打开斜杠菜单 | 中 |
| P1 | 工具栏溢出按钮(...),多出的项收进下拉 | 小 |
| P1 | 斜杠菜单里加状态标签 / 数学公式(如果团队需要) | 小 |
| P2 | Markdown 快捷键提示气泡(敲 `## ` 时浮一行"已转为 H2") | 极小 |

## 模块 7:附件

### 现状

后端两步上传(`/upload-url` 预签 → `/finalize` 校验大小落库),对象存 MinIO,数据库记元数据(`apps/api/src/routes/attachments.ts`)。MIME 白名单见 `packages/shared/src/constants.ts` 的 `ALLOWED_MIME_TYPES`,涵盖图片、PDF、Office、压缩包、纯文本/Markdown/CSV。

前端的插入入口:工具栏「插入图片/附件」按钮、斜杠菜单里"图片 / 附件"、粘贴图片、拖文件入编辑器。

`ImageAttachmentView.vue` 渲染:图片 → figure + 可编辑说明;非图片文件 → icon + 文件名 + 大小 + 下载按钮。

读视图有 `AttachmentLightbox` 大图预览。下载链接是 `GET /api/attachments/:id/raw`。

### 问题

- **没有页面级"附件面板"**。要把这页所有附件列出来,得滚文档找。Confluence 在右栏专门有个 "Attachments" 区块。
- **不支持多文件粘贴**(只取第一个),拖拽也类似(虽然代码看起来都支持,但 UI 没给出多文件进度)。
- **没有上传进度 UI**:finalize 是 fire-and-forget,大文件体感"按了没反应"。
- **没有取消上传**,大文件传错只能等它跑完。
- **删除文档里的附件节点 ≠ 删除对象**:除非走"替换"路径,inline 删除只删节点,MinIO 里那个孤儿文件会一直留着。
- **没在 UI 上提示每个文件的单文件上限**(后端有,前端不暴露)。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 阅读/编辑视图右栏加 "本页附件" 面板(列名/类型/大小/上传人/下载/删除) | 中 |
| P0 | 多文件粘贴 + 多文件上传队列(可见进度,可取消) | 中 |
| P1 | inline 删除节点时,弹"是否一并删除对象存储里的文件?" | 小 |
| P1 | 上传前/中显示单文件上限提示 | 极小 |
| P2 | 附件面板支持拖拽排序 | 小 |
| P2 | 后台定期清理孤儿 S3 对象(按引用扫描) | 中 |

## 模块 8:阅读视图与页面信息

### 现状

`ReadView.vue` 是一个独立组件(不是把编辑器 hidden),渲染服务端存的 `contentHTML`(`v-html` + `sanitizeAndHardenLinks` 防 XSS + `collapseTogglesByDefault`)。

右栏 `TocPanel.vue` 用 `IntersectionObserver` 做 scroll-spy + 点击锚点跳转;`HeadingAnchor` 在标题旁挂锚点链接。
页面 hover 内部链接 500ms 弹 `PageLinkPreview.vue` 预览卡。
日期节点(`DateInline`)每 60 秒重算一次相对时间。
代码块读视图里再走一遍语法高亮。

页面顶部操作区:导出菜单(单页 HTML/MD/PDF)、页面历史、关注按钮、编辑按钮。
Byline:最后编辑者头像 + 名字 + "最后编辑于 {相对时间}" + 字数。
Byline 上方有 "已发布" 状态标签(永远为真)和 "由 X 创建"(仅当创建者 ≠ 编辑者)。
底部:labels / 点赞头像行 / 子页列表 / 评论区。

### 问题

- **没有"页面信息"侧栏**:版本数、创建时间、关注者数量、权限限制——这些信息散在历史页/关注按钮/版本里,没一个地方能一眼看完。Confluence 有 "Page Information" 对话框。
- **没有反向链接面板**(哪几页 PageRef 链到本页),作者想看"谁引用了我"得手动搜。
- **没有阅读时长估算**(只有字数)。
- **Byline 头像没有 hovercard**:鼠标移到头像上不显示用户基本信息/联系方式。
- **没有打印按钮**:`styles/print.css` 存在但没暴露入口,真要打印只能浏览器原生打印(样式可能不友好)。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 右栏加"页面信息"小卡:版本数 / 创建时间 / 关注者头像组 / 限制说明 | 中 |
| P0 | 新增"反向链接"面板(PageRef 入向聚合),放在历史旁 | 中 |
| P1 | byline 加阅读时长估算 | 极小 |
| P1 | byline 头像 hovercard(显示用户信息) | 小 |
| P2 | 阅读视图加打印按钮(复用 print.css) | 极小 |

## 模块 9:页面元数据 / 标题 / 状态

### 现状

标题就是一个 `<input class="edit-title-input">`,进入编辑自动聚焦,失焦/提交时用 `normalizeTitle` 规范化(去首尾空白等)。默认占位提示用 `DEFAULT_TITLE` 常量。

labels:`<LabelPills>` 在 ReadView/EditView 内容底部展示,有 `LabelAddPopover` 加新 label。

保存状态指示:EditView 顶部有 `idle/pending/saving/saved/error` 五态 pill,只反映自动保存的状态。**没有"草稿/已发布"概念**:个人空间是天然草稿区,作者在团队空间里写的每页都是协作/实时生效,没有"先藏着"的需求。
"已发布" pill 在阅读视图顶部**永远显示**,其实没意义(团队空间页就是默认已发布,装饰而已)。

页面没有 emoji 图标、没有封面图、没有标题 slug 字段、没有 sibling 排序的 UI 入口(只能拖)、没有"限制可见性"开关(只能靠空间 + 组授权)。

### 问题

- **没有页面 emoji + 封面图**(Confluence/Notion 都允许),页与页之间只能靠标题区分,扫一眼空间很难认。
- **"已发布" pill 是死 UI**:永远为真,在团队空间页上是冗余装饰;在个人空间页上不该出现(那本来就是私人的)。要么按空间条件渲染,要么直接拿掉。
- **labels 在 pill 上不能 inline 删除**(只能开 popover 删),操作不直接。
- **标题 slug 不在 URL 里**(详见模块 4)。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 加页面 emoji(可空,默认从标题派)+ 封面图(从附件选) | 中 |
| P1 | label pill 上加 inline 删除 X(hover 才出) | 极小 |
| P1 | "已发布" pill 改成"按空间条件显示"(只在团队空间页上显示,代表"这是团队可见")或直接拿掉 | 极小 |
| P2 | URL slug 化(联动模块 4) | 中 |

## 模块 10:版本 / 历史 / 快照

### 现状

后端有三条相关路由:

- `GET /api/pages/:id/versions` 分页倒序
- `POST /api/pages/:id/snapshots` — **唯一的 checkpoint 入口**(EditView 30 秒 idle + 路由离开时自动触发)
- `POST /api/pages/:id/versions/:n/restore` 一键还原,自动写一条新版本

前端 `/p/:id/history`(`HistoryView.vue`)三栏:侧栏 / 版本时间线 / diff。`VersionDiffView.vue` 做**字符级 inline diff**(`textDiff.ts`),顶上显示 +N/-N 字符数。默认选最近两个对比。

保留 30 行硬编码(`page_versions` retention)。Restore 自动写入的 `changeNote` 是 "restored from v{N}",其他自动快照没有注释。

### 问题

- **只能对比相邻版本**:Diff 永远拿版本 N 和 N-1 比,不能任选两版。Confluence 支持任意两版并排对比,这是高级用户的刚需。
- **30 行保留硬编码**(trash retention 是 admin 可配的,历史不是,不对称)。
- **自动快照没法起名字**:history 一片"自动快照",用户根本看不出哪个是哪个。`SnapPageInputSchema.changeNote` 字段后端支持,前端没暴露。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 版本列表加复选,支持任选两版做 diff | 中 |
| P0 | 编辑视图退出/idle 时弹"保存为命名版本?"(让用户填 changeNote) | 中 |
| P1 | 保留数可配(挪到 admin setting,跟 trash retention 一致) | 小 |
| P2 | 版本列表加按作者筛选 + 关键字搜 changeNote | 小 |

## 模块 11:关注与通知

### 现状

后端 `POST/DELETE /api/pages/:id/watch`(toggle,带 watchersCount 返回)+ `GET /:id/watchers` 分页。`notify.ts` 的 `enqueueWatchFanout` 在 5 个事件触发:**page_edit / page_renamed / page_moved / page_restored / page_deleted**,5 分钟内同 actor 同 kind 去重。

前端 `PageWatchButton.vue` 在 ReadView 顶部,显示 watchers 数量(>0 才出),点一下 toggle。`NotificationBell.vue` 是个 440px 右抽屉,带 tab(全部/未读),30 秒轮询未读数。`/me/watched` 列所有被关注页。

**Schema 错位**:`NotificationKind` zod schema 只定义了 `mention / reply / comment_on_my_page / page_like` 4 个值,**但服务端 fanout 实际写的 kind 是上面那 5 个**。结果:改名/移动/还原/删除触发的通知写进了 DB,但前端枚举映射里没这几项,渲染时落到 "通知" 这个 fallback 文案。

### 问题

- **枚举错位导致通知文案丑**:你关注的页被人改名/移动,通知抽屉里只显示"通知",点进去才看到详情。
- **没有"空间级关注"**:我关注一个空间,所有页变更都推给我;目前只能挨个加页。
- **没有"关注者头像列表"**:ReadView 只在 watchers >0 时显示一个小数字 chip,看不到是谁。
- **用户不知道关注会推什么**:按钮 tooltip 没说明触发条件,用户会困惑"我怎么没收到通知"。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 修 `NotificationKind` schema,补齐 `page_renamed/moved/restored/deleted`,前端 HUMAN_KIND 加映射 | 小 |
| P0 | ReadView 顶部加关注者头像组(超过 X 个折叠) | 小 |
| P1 | 加空间级 watch(空间首屏有 toggle) | 中 |
| P1 | watch 按钮 tooltip 列出触发事件类型 | 极小 |
| P2 | 通知抽屉加"按 kind 筛选" | 小 |

## 模块 12:权限 / 可见性 / 移动 / 复制 / 归档 / 删除

### 现状

权限模型很简单:**空间级**。`canAccessSpace(userId, isAdmin, spaceId)` 路由级校验 + `assertAdminNotWritingPersonalSpace` 守卫。**没有页面级权限**,可见性 = "你能进这个空间就能读/编辑/软删"。硬删(`?purge=true`)和还原是 admin 专属。

个人空间每个用户一个,通过专属组 `pg-<userId>` 隔离,默认只对自己可见;管理员能浏览(运维/找回内容),作者可主动"发布到团队空间"让其他人看(走 `POST /:id/publish`)。个人空间不需要"分享给特定人"路径。

`PATCH /:id/move`:循环检测 + 个人空间禁写 + 跨空间仅允许从个人空间出发。拖树用 `usePageTreeDrag` 组合式。
`POST /:id/duplicate`:默认浅复制(标题加"复制自"前缀),`?withChildren=true` 深复制。labels/watch 不继承。
`POST /:id/publish`:个人 → 团队,创建新页。
`POST /:id` 软删:空间成员都能用,**有未删除子页时返回 409**,要求先删子页。`/manager/trash` 是 admin 专属回收站,支持批量还原/清空,`trash_retention_days` 可配(预设 0/7/30/90/180)。

### 问题

- **没有"归档"独立于"回收站"**:Confluence 归档的页**还在树里**(标灰),只是默认搜索排除;power-wiki 软删就直接从树消失,要恢复得去 admin 回收站。日常"先归档别再看到"的需求没出口。
- **没有"页面级限制"覆盖**:无法让某个页只对某几个组可见,空间是粒度天花板。
- **复制"带子页"开关没 UI**:`ReadView.vue` 的 onDuplicate 处理里这段被注释掉了。
- **软删带子页 409,没批量删子页的引导**:用户卡在"得先手动把子页一个个删掉"的死循环里。
- **跨空间移动 OK,但同空间移动没有"X 把这页从 A 移到 B" 的指示**:`page_event.moved` payload 带了 fromParentId/toParentId,ReadView 没读。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 加"归档"状态(独立于回收站):页保留在树里标灰,默认搜索排除 | 中 |
| P0 | 复制对话框加"是否包含子页"开关(后端已支持) | 极小 |
| P1 | 加"页面级限制"覆盖(选组,空间成员 + 限制 = 实际可见) | 中 |
| P2 | 软删时若带子页,弹"批量删除子页"二次确认 | 小 |
| P2 | ReadView byline 显示"X 把这页从 A 移到 B"指示 | 小 |

## 模块 13:活动流 / 标签

### 现状

**活动流**:`GET /api/pages/activity`(v2 版)记录 8 种事件——created / edited / moved / restored / duplicated / published / trashed / purged,`recordPageEvent` 在所有写路径调用。**50 条/页硬上限**。`ActivityView.vue` 在 `/activity`,有空间下拉(含"所有空间"选项)+ 彩色 kind chip + 点击跳页。

注意:comment **不**进 `page_event`(符合 CLAUDE.md 硬约束),但 `page_like` **会**(因为它在通知路径里被扇出)——两边不对称。

**标签**:`page_labels` 全局唯一表(没空间隔离,不分层,没命名空间)。`POST /api/labels` 加(幂等,trim+小写归一,最长 32 字符),`DELETE` 删,`GET /api/labels/search?q=` 搜索。`<LabelPills>` 在 ReadView/EditView 内容底部共享 reactive 数据,top search bar 支持 `?label=` chip 过滤。

### 问题

**活动流**:

- 只能按空间过滤,**不能按 kind 过滤**(想只看"删除事件"做不到)、**不能按操作人**、**没有日期范围**。
- **没有单页活动时间线**:"这页发生过什么"只能去全工作区 feed 翻。
- comment vs page_like 不对称:长讨论流不产生活动,但一次点赞会产生。

**标签**:

- **没有"所有标签"总览页**(像 Confluence 的 `/labels` index),你想看"团队都用了哪些标签"只能搜。
- **没有标签使用计数**展示在补全里(补全给你一个标签,你不知道这标签已经用在多少页)。
- **没有命名空间/前缀机制**:`team:design` 就被存成字符串 `team:design`,没法按 `team:` 一键筛出团队所有标签。
- **没有管理员重命名/合并标签**的能力(typo 标签只能一个个改回去)。

### 建议方向

| 优先级 | 方向 | 大致工作量 |
|---|---|---|
| P0 | 活动流加按 kind / 操作人 / 日期范围筛选 | 中 |
| P0 | 单页活动时间线(HistoryView 加 "活动" tab,与版本并列) | 中 |
| P1 | 建 `/labels` 总览页(标签云 + 每标签的页数,点击跳结果列表) | 中 |
| P1 | 标签补全显示使用计数 | 极小 |
| P2 | 标签命名空间约定(`team:` / `status:` 等保留前缀),管理员可重命名/合并 | 中 |
| P2 | 修 page_like vs comment 不对称(comment 也算活动 OR 把 like 从活动移除,二选一并明示规则) | 极小 |

## 几点全局说明

1. **本份文档不附实现代码**——后续按优先级走实现计划,每个建议方向可以单独排期,单点改动通常 ≤ 1 天。
2. **优先级可调**:你看完这份后,如果觉得某个模块的 P0 其实是 P2,某个 P2 其实卡你痛点,直接说,后续按你的口径排。
3. **跨模块联动**已经在备注里点了(slug↔模块 4,kind 枚举↔模块 11),实现时建议按"先底层后上层"的顺序推进(比如先把状态字段、slug、kind 枚举这些数据契约改了,再做 UI 套上去)。
4. **本次审视严格遵守 CLAUDE.md 硬约束 + 你的产品口径**:未提议暗色模式、移动端适配、协同编辑、外部图片、全文检索、Comment 优化、Reaction 表情、空间总览页、个人空间分享、草稿/已发布切换。
5. **本份文档是产品审视**,不替代实现代码 —— 后续按优先级走实现计划时,先做数据契约(slug/状态字段/枚举对齐),再做 UI 套用。
